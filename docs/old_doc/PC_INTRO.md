## 1. コードを書くということ

私たちが、日常、使っている言語、つまり日本語や英語は「**自然言語**」と呼ばれます。プログラミング言語は、人工的に作られた言語であるのに対し、自然に発生してきた言語だからです。

**自然言語**
- 英語、日本語、...

**人工言語**
- プログラミング言語、SQL、数式

コードを書くということは、自然言語で文章を書くことはどう違うのか？ これを理解するのはとても大切です。なぜなら、生成AIが普及してくると、プログラムを書くといっても、生成AIに対して自然言語に指示を書くことが多くなるからです。

---

## 2. そもそも、自然言語でプログラミングしないのか？

自然言語は、ひと言でいうと「**あいまい**」です。

たとえば、アウトレット店に入るとよく見かけるセールスを思い出してみましょう。

**「20％引き。1万円以上購入なら、さらに10%引き」**

もし2万円購入したとき、結局、いくら割引されるのでしょうか？

- **解釈①** 合計30%引き
- **解釈②** 20%引いた後の価格から、10%引き（合計28%引き）

自然言語は、人々の会話の中から自然に発展してきたため、誰かが明確に決めた文法や解釈があるわけではありません。だから、どちらの解釈をしても間違っているともいえません。だからあいまいなのです。

だから、買い物をする前に、本当の意味を確認するしかありません。

もしこれがコンピュータに与える指示だった場合、コンピュータもどちらの解釈をしたらいいのか明確に決められません。ひどいときは、毎回、ランダムに解釈が変わります。

**これでは困ります！**

プログラミング言語は、どんな言語であっても、書かれたコードの解釈が常に一つになるように、文法や書き方を決められています。コンピュータは、その文法と解釈にしたがって、どのコンピュータが処理しても同じ結果になるようになっているわけです。

これが生成AIの時代になっても、本質的にプログラミング言語がなくならない理由です。

どうやって厳密に指示を与えているか、Yuiを使いながら見ていきましょう。

## 3. 変数: データの記録場所

プログラミング言語は、データの記憶場所に対して名前をつけられるようになっています。

```yui
x = 42
```

上の例では、`x` と名づけたデータの記憶場所に対して、`42` という数値データを記憶させていることになります。

記憶場所は、コンピュータ上のどこかにあるのですが、プログラミングではそれは考えなくても良いようになっています。

その代わり、一度、名づけた名前を呼べば、いつでもその値を取り出すことができます。

```yui
x = 42

>>> x
42
```

**Yuiは、変数名を書くと実行結果を出力します**

この仕組みは、数学の変数によく似ているので、**変数**と呼ばれます。ただし、数学みたいに `x`, `y`, `z` のような一文字だけでなく、`weight` や `体重` のようなちゃんとした名前を付けられます。

```yui
体重 = 65

>>> 体重
65
```

⚠️ **重要**: `体重` と名前を付けたからといって、特別な意味や解釈が加わることはありません。`x` と名づけても `体重` と名づけても、コンピュータ上のどこかのデータの記憶場所に名前を付けただけです。

コンピュータにとってはどんな名前を付けようと同じですが、人間にとっては名前はとても重要です。意味を思い出しやすい名前にしておくと、コードを読んだとき、何の処理をしようとしているのか把握しやすくなります。

---

## 4. コンピュータは"よしな"に解釈しない

変数を使うときの規則は、最初に人間側がデータに名前をつけることです。コンピュータが勝手に名前をつけることはありません。たぶん、こういう名前だろうと憶測したりしません。

もし一度もデータが与えられていない変数を使ったり、もっと単純に、名前を書き間違えたりすると、変数名からデータが探せないのでエラーになります。

```yui
x = 42

>>> X
エラー: 変数 'X' が見つかりません
```

**`x` の代わりに `X` を使ってみるとエラーになる**

上の例は、`x` の代わりに大文字の `X` で値を呼び出そうとしているのですが、見つからないからエラーになっています。

コンピュータは、できないことは、知らないことは、はっきりとエラーとして返してきます。人間のように"たぶん"とか、"良しなに"解釈してくれません。

## 5. 配列: 複雑なデータ

コンピュータでは、複雑な情報は数値を並べた**配列**として扱います。たとえば、7個、数が並んだデータは次のように、`[ ]` で囲んで書きます。

```yui
a = [1, 2, 3, 4, 5, 7, 8]

>>> a
[1, 2, 3, 4, 5, 7, 8]
```

**多くのプログラミング言語に共通する記法です**

このような配列の中身にアクセスするときは、先頭から数えて何番目か**添え字**で指定します。このとき、**0から数えます**。

```yui
a = [1, 2, 3, 4, 5, 7, 8]

>>> a[0]
1
>>> a[1]
2
>>> a[6]
8
```

**配列の先頭は 0 番目**

配列の添え字:
- `a[0]` → 最初の要素（1）
- `a[1]` → 2番目の要素（2）
- `a[6]` → 7番目の要素（8）

もし配列の長さを超えてしまうと、エラーになります：

```yui
a = [1, 2, 3, 4, 5, 7, 8]

>>> a[10]
エラー: インデックスが範囲外です
```

配列の長さは `|配列|` で取得できます：

```yui
a = [1, 2, 3, 4, 5, 7, 8]

>>> |a|
7
```

## 6. 文字列

Yuiでは、文字列は**文字コードの配列**として扱います。

```yui
s = "Hello"

>>> s
"Hello"
>>> s[0]
72
>>> s[1]
101
```

文字コードは、英字がASCIIコード、それ以外はユニコードです。

主な文字コード:
- `'A'` → 65
- `'Z'` → 90
- `'a'` → 97
- `'z'` → 122
- `'0'` → 48
- `'9'` → 57
- `' '`（スペース） → 32

```yui
text = "ABC"

>>> text[0]
65
>>> text[1]
66
>>> text[2]
67
```

🗒️ **練習**: `"Hello"` の各文字の文字コードを調べてみよう

## 7. Yuiの計算

Yuiは、変数 `x` に格納された値に対して、次の2種類の計算ができます：

- `xを増やす`
- `xを減らす`

**「xを増やす」を実行すると、xの値が1つ増えます。**

```yui
x = 5
xを増やす

>>> x
6
```

もし2つ増やしたいときは、次のように「xを増やす」という操作を2回書きます。

```yui
x = 5
xを増やす
xを増やす

>>> x
7
```

自然言語なら「xを二つ増やす」みたいに書けますが、プログラミングでは決められた書き方しかできません。決められた書き方を工夫します。

もし日本語で「体重を増やす」と書いてしまったら、いろいろな意味が発生しますし、状況によっては失礼な発言になってしまいます。しかし、Yuiで「体重を増やす」と書いたら、体重は変数名に過ぎず、その値をひとつ増やす以外の解釈はありえません。

```yui
体重 = 60
体重を増やす
体重を増やす
体重を増やす

>>> 体重
63
```

**クイズ**: 体重はいくつになるでしょうか？ → **答え: 63**

配列の中身も正しく添え字を指定すれば、増やしたり、減らしたりできます。

```yui
数列 = [10, 20, 30, 40, 50]
数列[0]を増やす
数列[2]を減らす
数列[4]を増やす
数列[4]を増やす

>>> 数列
[11, 20, 29, 40, 52]
```

**クイズ**: 数列の中はどうなりますか？ → **答え: [11, 20, 29, 40, 52]**

文字列も文字コードの配列なので、同じように操作できます：

```yui
s = "ABC"
s[0]を増やす

>>> s
"BBC"
```

**文字列sはどうなりますか？** → **答え: "BBC"** （'A'のコード65が66='B'になる）

### 標準ライブラリを使った計算

Yuiには、もっと便利な計算機能があります。ファイルの先頭に `標準ライブラリを使う` と書くと、算術関数が使えるようになります。

```yui
標準ライブラリを使う

x = 和(10, 20, 30)

>>> x
60
```

主な算術関数:
- `和(a, b, c, ...)` - 足し算
- `差(a, b)` - 引き算
- `積(a, b, c, ...)` - 掛け算
- `商(a, b)` - 割り算
- `剰余(a, b)` - 余り

```yui
標準ライブラリを使う

合計 = 和(10, 20, 30)
差 = 差(100, 35)
積 = 積(5, 6, 7)

>>> 合計
60
>>> 差
65
>>> 積
210
```

## 8. くり返し

何度も同じような処理を指示するのは大変です。プログラミングの重要な機能のひとつに同じ処理を「**くり返す**」ことがあります。

Yuiでは、「xを増やす」を5回、くり返したいときは次のようにくり返したい処理を `{ }` で囲んで書きます。

```yui
x = 0
5回、くり返す {
    xを増やす
}

>>> x
5
```

**クイズ**: x はいくつになりますか？ → **答え: 5**

急にコードっぽくなりました。コードを書いている感を味わえるように、JavaやJavaScript などでもおなじみの `{ }` (← カーリーブレイスと読む) を使っています。`{` と `}` で囲むことで、くり返す範囲を指定しています。

**重要**: Yuiでは、すべての `{` の後には**必ず改行**が必要です。

```yui
x = 0
10回、くり返す {
    xを増やす
    xを増やす
}

>>> x
20
```

**クイズ**: xはいくつになりますか？ → **答え: 20**

`{ }` の中には、このように好きなコードを書くことができ、上から順番に実行されます。`{ }` 内のプログラムが読みやすくするように字下げ（インデント）すると良いでしょう。

```yui
x = 1
5回、くり返す {
    xを増やす
    xを増やす
    xを減らす
}

>>> x
6
```

**クイズ**: 結局 xの値は何になるのでしょうか？ → **答え: 6**
（5回くり返すたびに、+2-1=+1 されるので、1+5=6）

## 9. ルールを書こう

ルールとは、「もし〜 ならば」のように条件に対する処理を書く方法です。英語のIF-THENに相当するので、IF-THENルールとも呼ばれます。

```yui
x = 5
もし x が 10 より小さい ならば {
    xを増やす
}
そうでなければ {
    xを減らす
}

>>> x
6
```

「そうでなければ」の部分は、省略することができます。

```yui
x = 5
もし x が 0 以外 ならば {
    xを増やす
}

>>> x
6
```

Yuiでは、「xが0ならば」の部分に比較の方法を書くことができます。

比較の書き方:
- `もし x が 0 ならば、` - 等しい
- `もし x が 0 以外 ならば、` - 等しくない
- `もし x が 0 以上 ならば、` - 以上
- `もし x が 0 以下 ならば、` - 以下
- `もし x が 0 より大きい ならば、` - より大きい
- `もし x が 0 より小さい ならば、` - より小さい

⚠️ 「もし体重が太り過ぎならば」のような自由に書けません。決められた文法の通りしか書けません。

実際、次のコードは、Yuiの文法的に100%正しいコードですが、Yuiは `偶数` というのはデータの記憶場所の名前に過ぎず、`x` が2と等しいか判定しているに過ぎません。

```yui
x = 4
偶数 = 2
もし x が 偶数 ならば {
    xを増やす
}

>>> x
4
```

⚠️ **重要**: `偶数` というのは、0, 1, 2 というような具体的な数に比べて抽象度の高い概念なんです。だから、変数と偶数を直接比較できません。

もちろん、Yuiでも偶数か判定するルールも書くことはできます。それは次の「関数」のセクションで学びましょう。

## 10. 関数: プログラムの単位

関数は、プログラミングにおいて、プログラムの機能を実現する基本単位です。数学の関数に似ていますが、少し異なります。

```
┌─────────┐
│  入力   │
└────┬────┘
     │
     ▼
┌─────────┐
│  関数   │  ← プログラム
└────┬────┘
     │
     ▼
┌─────────┐
│  出力   │
└─────────┘
```

生成AIによるプログラミングでも、コードを関数単位で見るのは重要です。なぜなら、生成AIといっても、一度に生成できるコード量には上限があって、大きなコードは関数単位で分割することになるからです。

まず簡単な関数を書いてみましょう：

```yui
f = 入力 x に対し {
    xを増やす
    xが答え
}

>>> f(0)
1
>>> f(5)
6
>>> f(10)
11
```

**関数名は f**

関数は、入力に対して何かしら処理して出力するプログラムの単位です。

一度関数を定義すると、`f(0)` や `f(9)`, `f(f(0))` のように自由に組み合わせて、プログラムを呼び出して結果を得ることができます。

```yui
f = 入力 x に対し {
    xを増やす
    xが答え
}

>>> f(f(0))
2
>>> f(f(f(0)))
3
```

### 練習問題

#### 1. 絶対値の関数を作ってみよう

```yui
絶対値 = 入力 x に対し {
    もし x が 0 より小さい ならば {
        result = 0
        result = 差(result, x)
        resultが答え
    }
    そうでなければ {
        xが答え
    }
}

>>> 絶対値(-5)
5
>>> 絶対値(3)
3
>>> 絶対値(0)
0
```

関数の本体内部で、ルール（条件分岐）を使って関数を定義します。

#### 2. たし算の関数を作ってみよう

たし算は、増やすと繰り返しを使って作ります。手順を考えるのがプログラミングです。

```yui
たし算 = 入力 x, y に対し {
    z = 0
    x回、くり返す {
        zを増やす
    }
    y回、くり返す {
        zを増やす
    }
    zが答え
}

>>> たし算(3, 5)
8
>>> たし算(10, 20)
30
```

**「zが答え」を忘れないように**

新しい変数 `z = 0` を使わなくても作ることができます。

```yui
たし算 = 入力 x, y に対し {
    y回、くり返す {
        xを増やす
    }
    xが答え
}

>>> たし算(3, 5)
8
```

**今度は「xが答え」**

このように同じ答えを出すための手順は色々あります。同じようにプログラムの書き方も色々あります。

**プログラムの正しさ**
- 入力に対して正しい出力がされれば正しい

#### 3. かけ算の関数を作ってみよう

かけ算は、九九表を覚えて計算しました。九九表と同じくルールで書くことができます。

しかし、全部書き出すと大変です：

```yui
かけ算 = 入力 x, y に対し {
    result = 0
    もし x が 1 ならば {
        もし y が 1 ならば { result = 1 }
        もし y が 2 ならば { result = 2 }
        # ...とても長くなる
    }
    # ...
    resultが答え
}
```

たし算のときと同様に、くり返しを用いて考えましょう。

- `x × 1 = 0+x` （0にxを1回足す）
- `x × 2 = 0+x+x` （0にxを2回足す）
- `x × 3 = 0+x+x+x` （0にxを3回足す）
- `x × y = 0+x+x+x+...+x` （0にxをy回足す）

たし算をくり返すことでかけ算も作れます。

```yui
かけ算 = 入力 x, y に対し {
    result = 0
    y回、くり返す {
        result = たし算(result, x)
    }
    resultが答え
}

>>> かけ算(3, 4)
12
>>> かけ算(7, 8)
56
```

**かけ算の中から、たし算を呼び出す**

かけ算の中からたし算を呼び出しているように、一度定義した関数を呼び出すことでより複雑なプログラムを書くことができます。

### 標準ライブラリを使った実装

実は、Yuiには標準ライブラリがあるので、もっと簡単に書けます：

```yui
標準ライブラリを使う

たし算 = 入力 x, y に対し {
    和(x, y)が答え
}

かけ算 = 入力 x, y に対し {
    積(x, y)が答え
}

>>> たし算(10, 20)
30
>>> かけ算(7, 8)
56
```

## 文法をしっかり学ぶ

プログラミングは、Yuiに限らず、文法をしっかり覚えて、ちゃんとコードをコンピュータが理解できるように書けるようになる必要があります。

高校の情報Iに登場するプログラミング言語は、Pythonにしても、JavaScriptにしても、英語がベースになっていてかなり複雑です。文法を覚えるのにかなりの時間が必要です。

例えば、同じ処理をJavaScriptで書くと：

```javascript
// JavaScript
let x = 0;
for (let i = 0; i < 10; i++) {
    x = x + 2;
    x = x - 1;
}
console.log(x);  // 10
```

```yui
# Yui
x = 0
10回、くり返す {
    xを増やす
    xを増やす
    xを減らす
}

>>> x
10
```

Yuiの文法は、日本語をベースにして、コンピュータに指示を伝えるべき、最小限の文法しかありません。基本的に今日習ったことを組み合わせてプログラムを書きます。

### Yuiの構文規則（簡略版）

**文の種類:**
- `変数 = 式` （変数定義）
- `変数を増やす` （インクリメント）
- `変数を減らす` （デクリメント）
- `変数の末尾に 式 を 追加する` （配列への追加）
- `もし 式 が 式 [比較] ならば { 文* } そうでなければ { 文* }` （条件分岐）
- `式回、くり返す { 文* }` （くり返し）
- `式が答え` （関数の戻り値）
- `くり返しを抜ける` （break）
- `>>> 式 期待値` （テストケース）

**式の種類:**
- `変数` （変数参照）
- `整数値` （整数リテラル）
- `浮動小数点数` （小数リテラル）
- `"文字列"` （文字列リテラル）
- `-式` （マイナス）
- `[式, ...]` （配列リテラル）
- `|式|` （配列の長さ）
- `関数名(式, ...)` （関数呼び出し）
- `入力 変数, ... に対し { 文* }` （関数定義）

**比較演算:**
- （なし） - 等しい
- `以外` - 等しくない
- `以上` - 以上
- `以下` - 以下
- `より大きい` - より大きい
- `より小さい` - より小さい

## まとめ

Yuiを使って、プログラミングの基本概念を学びました：

1. **変数** - データに名前をつける
2. **配列** - 複数のデータをまとめて扱う
3. **文字列** - 文字コードの配列として扱う
4. **計算** - `xを増やす`、`xを減らす`、標準ライブラリ関数
5. **くり返し** - 同じ処理を繰り返す
6. **条件分岐** - 条件によって処理を変える
7. **関数** - 処理をまとめて再利用する

これらの概念は、どのプログラミング言語でも共通する重要な要素です。Yuiで学んだことは、他の言語を学ぶときにも役立ちます。
