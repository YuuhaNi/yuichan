# Yui ハマりどころ集

## 1. 『もし』の条件に真偽値を直接置けない

### 症状

`もし` の条件部に、真偽値を返す式をそのまま置くとパース失敗する。

```
もし整数判定(x)ならば{ ... }       ← FAIL
もしokならば{ ... }                ← FAIL  (ok は bool 変数)
```

### 回避策

`もし A が B <接尾辞> ならば` の中置比較で、真偽値なら `真` と比較する。

```
もし整数判定(x)が真ならば{ ... }
もしokが真ならば{ ... }
```

### ルール

yui の `もし` は **中置比較必須**。python の `if x:` の感覚で真偽値を
直接書かず、必ず `が 真 ならば` / `が 偽 ならば` を明示する。

## 2. 比較の接尾辞は 5 種類だけ

### 症状

日本語として自然な比較語を書いてもパース失敗する。

```
もしnが2未満ならば{ ... }             ← FAIL  (「未満」は未対応)
もしaがbでないならば{ ... }            ← FAIL  (「でない」は未対応)
もしsが"x"を含むならば{ ... }          ← FAIL  (該当する接尾辞なし)
もしaがbと等しくないならば{ ... }       ← FAIL
```

### 回避策

使える接尾辞は以下の 5 種類のみ。

| 比較 | 接尾辞 |
|------|--------|
| `==` | (なし) |
| `!=` | `以外` |
| `<`  | `より小さい` |
| `>`  | `より大きい` |
| `<=` | `以下` |
| `>=` | `以上` |

```
もしnが2より小さいならば{ ... }
もしaがb以外ならば{ ... }
```

### ルール

「未満」「でない」「異なる」「を含む」はすべて自然な日本語に見えるが
yui では使えない。必ず上の表の接尾辞に置き換える。
「を含む」は組み込みがないので、自前で走査する必要がある。

## 3. 無限ループ構文はない

### 症状

回数を省略した `くり返す { ... }` はパース失敗する。

```
くり返す{
   もし条件ならば{ くり返しを抜ける }
   ...
}                                       ← FAIL
```

`i が x 以下の間くり返す` のような while 相当もない。

### 回避策

十分大きな上限を置き、条件を満たしたら `くり返しを抜ける` で脱出する。

```
100000回くり返す{
   もし条件ならば{ くり返しを抜ける }
   ...
}
```

### ルール

反復構文は `N回くり返す` のみ。`N` は必ず整数リテラルまたは変数で書き、
終了条件が事前に分からないときは **上限 + 早期脱出** で書く。

## 4. 非存在組み込み関数を呼ばない

### 症状

python や javascript 標準の関数名をそのまま呼ぶとパース後に未定義エラー。

```
arrをソートする                          ← FAIL
並べ替え(arr)                           ← FAIL
分割(s, ",")                            ← FAIL
"a,b"を","で分割                        ← FAIL
辞書のキー列                            ← FAIL
md5(s)                                  ← FAIL
切り上げ(x)                             ← FAIL
天井(x)                                 ← FAIL
べき乗(a,b)                             ← FAIL
除算(a,b)                               ← FAIL
approx_zero(...)                        ← FAIL
rshift(x, n)                            ← FAIL
文字列補間(...)                          ← FAIL
```

### 回避策

yui の組み込みは以下の 20 種のみ。それ以外は自前で実装する。

| 種別 | 関数 |
|------|------|
| 算術 | `和`, `差`, `積`, `商`, `剰余`, `絶対値`, `平方根`, `乱数` |
| 集約 | `最大値`, `最小値` |
| 変換 | `整数化`, `小数化`, `文字列化`, `配列化` |
| 型判定 | `整数判定`, `小数判定`, `文字列判定`, `配列判定`, `オブジェクト判定`, `ブール判定` |

ソート・反転・部分文字列検索・分割・切り上げ・べき乗・md5・ハッシュ関数
などは組み込みにない。必要なら関数の中で自前ループで書く。

```
# 並べ替えは二重ループの隣接交換で書く
# 分割は文字コード配列の走査で書く
```

### ルール

「python にあるから yui にもあるはず」と考えない。組み込みにない処理は
**関数内にインライン展開** して書く。

## 5. 配列追加は `配列 に 値 を追加する` の語順

### 症状

助詞が逆の語順で書いてしまう。

```
stackをargに追加する                    ← FAIL  (語順が逆)
```

### 回避策

```
stackにargを追加する
```

### ルール

`配列 に 値 を追加する` で固定。日本語としてはどちらの語順も自然に読めるが、
yui では語順が固定されている。

## 6. 判定関数の戻り値は `1` / `0` (整数)

### 症状

判定関数が `真が答え` / `偽が答え` を返すと、期待値 `1`/`0` と不一致になる。

```
is_prime=入力nに対し{
   ...
   真が答え                              ← 期待値は 1
}
```

### 回避策

判定結果は整数 `1` / `0` で返す。

```
is_prime=入力nに対し{
   result=0
   もし ... ならば{ result=1 }
   resultが答え
}
```

### ルール

「〜かどうか」「含まれているか」「等しいか」など **yes/no を返す関数は
必ず 1/0 の整数で返す**。`真` `偽` (真偽値) を返すと数値比較と不一致になる。

複数の判定を組み合わせるときは `積(a,b)` (AND)、`和(a,b)` が 0 より大きいか
で判定 (OR)。

## 7. プロンプト内の例示関数を直接呼び出さない

### 症状

プロンプトに `bubble_sort` や `reverse` や `find` や `unique_chars` のような
サンプル関数があっても、生成する関数からそれらを呼び出すと未定義エラーになる。

```
median=入力lに対し{
   sorted=bubble_sort(l)                ← FAIL  (定義を含めていないと未定義)
   ...
}
```

### 回避策

再利用したい処理は関数の中にインライン展開する。

```
median=入力lに対し{
   # ここに bubble_sort の中身を直接書く
   n=lの大きさ
   sorted=[]
   i=0
   n回くり返す{ sortedにl[i]を追加する; iを増やす }
   i=0
   n回くり返す{
      j=0
      差(差(n,i),1)回くり返す{
         a=sorted[j]; k=和(j,1); b=sorted[k]
         もしaがbより大きいならば{ sorted[j]=b; sorted[k]=a }
         jを増やす
      }
      iを増やす
   }
   sorted[商(n,2)]が答え
}
```

### ルール

プロンプトの例示は「参考」であって「共有ライブラリ」ではない。
生成する関数の中で何らかの処理を使いたい場合は **その関数内で再実装** する。
特にソート・反転・部分文字列検索はこのミスが起きやすい。

## 8. 算術記号 `+ - * / %` は使えない

### 症状

```
和=a+b                                   ← FAIL
```

### 回避策

標準ライブラリ関数を使う。

| やりたいこと | 書き方 |
|--------------|--------|
| `a + b`      | `和(a,b)` |
| `a - b`      | `差(a,b)` |
| `a * b`      | `積(a,b)` |
| `a / b` (整数商) | `商(a,b)` |
| `a % b`      | `剰余(a,b)` |

ループのインデックス計算 (`i+1`, `n-1`) も同様で、`和(i,1)` / `差(n,1)` と書く。

### ルール

算術演算子は一切使えない。必ず関数呼び出し形式で書く。

## 9. 複数値を返すときは配列 `[a, b]`

### 症状

タプル相当の戻り値をオブジェクトで返すと期待値と不一致になる。

```
sum_product=入力arrに対し{
   ...
   {"sum":s, "product":p}が答え          ← 期待値は [s, p]
}
```

### 回避策

配列で返す。受け取り側は `r[0]`, `r[1]` で取り出す。

```
sum_product=入力arrに対し{
   ...
   [s, p]が答え
}
```

### ルール

複数の値を返したいときは **常に配列 `[a, b, ...]` で返す**。
オブジェクト (`{"key": v}`) で返すと評価のタプル期待と一致しない。
プロンプト中のサンプルには配列/オブジェクト両方の返し方が出てくるが、
本ベンチマークでは配列を選ぶ。

## 10. 文字列 `"0"` と整数 `0` は別物

### 症状

数値を期待する関数に文字列を渡すと型エラー。

```
change_base(x, "2")                     ← FAIL  (基数に文字列)
decimal_to_binary("0")                  ← FAIL
```

### 回避策

- `"42"` → 整数: `整数化("42")`
- 整数 → `"42"`: `文字列化(42)`
- 数字1文字コード (`'0'`=48, `'9'`=57) → 整数: `差(c, 48)`
- 整数 (0..9) → 数字1文字コード: `和(n, 48)`

### ルール

yui の文字列は内部的に文字コードの配列。リテラル `"0"` と整数 `0` は別の型
なので、どちらを扱っているか常に意識する。
