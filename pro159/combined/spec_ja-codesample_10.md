# Yui 言語 仕様書（日本語）

Yui は、日本語による簡単な教育用プログラミング言語です。
プログラムを構成する最小限の要素（型・式・文）のみを備え、
小中学生でも自然な日本語で読み書きできることを目指しています。

---

## 1. 字句

ソースコードを構成する最小単位（トークン）の規則です。

| 要素 | 例 |
|------|----|
| 空白 | 半角スペース, タブ, 全角スペース `　` |
| 改行 | `\n`（文の区切り） |
| 行コメント | `#` または `＃` から行末まで |
| 識別子 | `[A-Za-z_][A-Za-z0-9_]*`、および `「…」` で囲む拡張名 |

文は改行で区切られる。1 行に 1 文を書くのが基本です。

識別子には日本語も使えます（例: `絶対値`, `最大値`, `標準ライブラリを使う`）。

---

## 2. 型システム

Yui の値はすべて以下の 7 種類いずれかの型を持ちます。

| 型 |  リテラル |
|----|----------|
| ヌル |  `値なし` |
| 論理値 |  `真` / `偽` |
| 整数 |  `42` |
| 小数 |  `3.14`（表示時は小数点以下 6 桁固定） |
| 文字列 |  `"あ"`, `"x={x}"` |
| 配列 |  `[1, 2, 3]` |
| オブジェクト |  `{"x": 1, "y": 2}` |

- 文字列は、内部的に**文字コード（int）の配列**として保持される。これにより文字列と配列は同じ操作（要素追加・長さ取得など）で扱える。
- 整数も同様に、内部的には**2 進数のビット配列（LSB が先頭）**として保持される。したがって整数にもインデックス・長さ取得・要素追加などの配列操作が暗黙に適用できる（例: `6` は `[0,1,1]` として扱え、`6[0]` は `0`、`6[1]` は `1`、`6の大きさ` は `3`）。この性質を使えば、ビット演算（NOT・AND・OR・XOR・シフト・加減算・2 の補数）を純粋な配列操作として実装できる。
- 配列等値比較は再帰的で、文字コード配列と文字列、ビット配列と整数は相互比較可能（例: `[72,105] == "Hi"` は真、`[0,1,1] == 6` は真）。

## 3. 式

値を生み出す構文要素です。式は他の式の中に入れ子にでき、最終的に 1 つの値へ評価されます。

### 3.1 リテラル

ソース中に直接書かれる値の表記です。

| 型 |  リテラル |
|----|----------|
| ヌル |  `値なし` |
| 論理値 |  `真` / `偽` |
| 整数 |  `42` |
| 小数 |  `3.14`（表示時は小数点以下 6 桁固定） |
| 文字列 |  `"あ"`, `"x={x}"` |
| 配列 |  `[1, 2, 3]` |
| オブジェクト |  `{"x": 1, "y": 2}` |

### 3.2 文字列補間

文字列の中に式を埋め込み、結果を文字列化して連結します。

`"{expr}"` で式を埋め込み可能。`{` `"` 自体を文字として入れたいときは `\{` `\"` とエスケープします（`\\`, `\n`, `\t` も使えます）。

```
name="ゆい"
msg="こんにちは、{name}さん"
```

### 3.3 長さ演算

配列・文字列(expr)の要素数を取得します。

`exprの大きさ`

### 3.4 単項演算

ひとつの式に作用する演算子です。

`-expr`（符号反転）。リテラルに直接付けて `y=-2` のように書けます。

### 3.5 二項演算

Yui では算術記号を使わず、関数呼び出しの形で書きます。

**重要**: yui は `+ - * / %` は**無効**。標準ライブラリの関数（`和`, `差`, `積`, `商`, `剰余`）を使う。

| 関数 | 意味 |
|------|------|
| `和(x,…)` | 合計（引数が配列なら展開） |
| `差(x,y,…)` | 差 |
| `積(x,y,…)` | 積 |
| `商(x,y)` | 商（int/int は床除算） |
| `剰余(x,y)` | 剰余 |

### 3.6 インデックス / プロパティ

配列やオブジェクトから要素を取り出します。取り出した場所に代入もできます。

- `a[i]` → 配列 a の i 番目の要素（0 から始まる）
- `O["x"]` → オブジェクト O の `"x"` プロパティ。キーは文字列を使う
- `aの大きさ` → 配列 a（または文字列 a）の要素数

```
A=[10,20,30]
A[1]=200         # 2 番目の要素を書き換え

O={"x":0,"y":0}
O["x"]=1         # プロパティに代入
```

### 3.7 関数適用

定義済みの関数を呼び出して値を得ます。

`f(a, b, c)` — 引数は `,` 区切り。

## 4. 文

プログラムの実行単位です。文は上から順に評価され、変数の更新や制御フローを行います。

### 4.1 代入

変数に値を結びつけます。同じ名前に再代入すると上書きされます。

```
x = 式
```

### 4.2 増減

数値変数を 1 だけ増やす／減らすための簡略構文です。

```
xを増やす
xを減らす
```

### 4.3 追加

配列の末尾に値を 1 つ加えます。

```
aにxを追加する
```

文字列も文字コードの配列なので追加できる。

### 4.4 条件分岐

条件が成り立つかどうかで実行する処理を切り替えます。

```
もしxがyならば {
   …
}
そうでなければ {
   …
}
```

そうでなければ省略可能です。

#### 比較の書き方（接尾辞形式）

`もし <式1> が <式2> <接尾辞> ならば` の形で、接尾辞によって比較の種類が変わります。

| 接尾辞 | 意味 | 例 |
|--------|------|----|
| （なし） | `==`（等しい） | `もしxが0ならば` |
| `以外` | `!=`（等しくない） | `もしxが0以外ならば` |
| `より小さい` | `<` | `もしrがbより小さいならば` |
| `より大きい` | `>` | `もしxが10より大きいならば` |
| `以下` | `<=` | `もしdistが1以下ならば` |
| `以上` | `>=` | `もしnが100以上ならば` |
| `のいずれか` | `in`（含まれる） | `もし"banana"がfruitsのいずれかならば` |
| `のいずれでもない` | `not in`（含まれない） | `もし"grape"がfruitsのいずれでもないならば` |

`==` `!=` `<` `<=` `>` `>=` などの**記号による比較は yui 構文では使えません**。必ず `が〜ならば` と接尾辞の組み合わせで書きます。

#### 多段分岐（elif 相当）

Yui には `elif` 専用のキーワードはありません。`そうでなければ` の中にさらに `もし` をネストして表現します。

```
もしfizzが0ならば {
   もしbuzzが0ならば {
      resultに"FizzBuzz"を追加する
   }
   そうでなければ {
      resultに"Fizz"を追加する
   }
}
そうでなければ {
   もしbuzzが0ならば {
      resultに"Buzz"を追加する
   }
   そうでなければ {
      resultにiを追加する
   }
}
```

### 4.5 繰り返し

ブロックを指定回数だけ実行します。途中で抜けるには `くり返しを抜ける` を使います。

```
N回くり返す {
   …
}
```

### 4.6 関数定義

入力（パラメータ）から答え（戻り値）を計算する手続きを定義します。

```
f = 入力 x, y に対し {
   …
   式 が答え
}
```

- パラメータなし: `入力なしに対し{ … }`
- 戻り値: `expr が答え`
- 明示的 void 戻り: `関数から抜ける`
- 関数内で定義した変数は**その関数ローカル**。外側からは見えません。
- **暗黙の戻り値**: `が答え` も `関数から抜ける` も書かずに本体が終わると、**ローカル変数をまとめたオブジェクト**が返されます。コンストラクタ風の使い方ができます。

```
point=入力x,yに対し{
   # 何も返さない → {x, y} のオブジェクトが返る
}
O=point(3,5)
>>> O["x"]
3
```

### 4.7 アサート

式の評価結果が期待値と一致することを検証する、テスト用構文です。Python の Doctest と同じ書式を採用しています。

```
>>> 式
期待値
```

### 4.8 インポート

外部の関数群を現在のプログラムから利用できるようにします。

`標準ライブラリを使う` — 標準ライブラリを現在の環境に登録。

### 4.9 何もしない

プレースホルダとして、処理を行わない文を書けます。

```
何もしない
```

### 4.10 式文（Print Expression）

代入や制御文に当てはまらない単独の式は、その評価結果が自動的に印字されます。
REPL 的な確認に便利です。

---

## 5. ブロック

複数の文をひとまとまりに扱う構文です。条件分岐・繰り返し・関数定義の本体として使われます。

- yui（明示デリミタ）: `{ … }`

## 6. 標準ライブラリ（主要関数）

`標準ライブラリを使う` と宣言すると、以下の関数群が呼び出せるようになります。

| 名前 | 機能 |
|------|------|
| `絶対値(x)` | 絶対値 |
| `平方根(x)` | 平方根（Float） |
| `乱数()` | 0〜1 の乱数 |
| `和`, `差`, `積`, `商`, `剰余` | 四則演算 |
| `最大値(…)`, `最小値(…)` | 最大・最小 |
| `配列化(x)`, `文字列化(x)`, `小数化(x)`, `整数化(x)` | 変換 |
| `ブール判定(x)`, `整数判定(x)`, `小数判定(x)`, `文字列判定(x)`, `配列判定(x)`, `オブジェクト判定(x)` | 型判定 |

---

```yui
標準ライブラリを使う
# 簡易スタックマシン
# 命令は [opcode, operand] の形の配列で表す。operand が不要な命令は 0 を入れる。
#
# 対応命令:
#   ["PUSH", n] -- スタックに n を積む
#   ["POP",  0] -- スタック頂上を捨てる
#   ["DUP",  0] -- スタック頂上を複製する
#   ["ADD",  0] -- 上 2 つを取り出して a+b を積む
#   ["SUB",  0] -- 上 2 つを取り出して a-b を積む
#   ["MUL",  0] -- 上 2 つを取り出して a*b を積む
#
# プログラム実行後、スタックの頂上を答えとして返す。
# (Yui の配列は要素削除ができないので、論理的なスタックの大きさを top で管理し、
#  pop は top を減らすだけにする)

run=入力programに対し{
   stack=[]
   top=0
   n=programの大きさ
   i=0
   n回くり返す{
      instr=program[i]
      op=instr[0]
      arg=instr[1]

      もしopが"PUSH"ならば{
         # 論理 top が配列実長より小さければ上書き、そうでなければ末尾に追加
         sz=stackの大きさ
         もしtopがszより小さいならば{
            stack[top]=arg
         }
         そうでなければ{
            stackにargを追加する
         }
         topを増やす
      }
      もしopが"POP"ならば{
         topを減らす
      }
      もしopが"DUP"ならば{
         t=差(top,1)
         x=stack[t]
         sz=stackの大きさ
         もしtopがszより小さいならば{
            stack[top]=x
         }
         そうでなければ{
            stackにxを追加する
         }
         topを増やす
      }
      もしopが"ADD"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=和(a,b)
         topを増やす
      }
      もしopが"SUB"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=差(a,b)
         topを増やす
      }
      もしopが"MUL"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=積(a,b)
         topを増やす
      }
      iを増やす
   }
   # スタックの頂上を結果として返す
   topを減らす
   stack[top]が答え
}

# テスト: 3 + 4 = 7
>>> run([["PUSH",3],["PUSH",4],["ADD",0]])
7
# テスト: (3 + 4) * 2 = 14
>>> run([["PUSH",3],["PUSH",4],["ADD",0],["PUSH",2],["MUL",0]])
14
# テスト: 10 - (2 + 3) = 5
>>> run([["PUSH",10],["PUSH",2],["PUSH",3],["ADD",0],["SUB",0]])
5
# テスト: DUP で 5 を複製して掛ける → 25
>>> run([["PUSH",5],["DUP",0],["MUL",0]])
25
# テスト: POP で途中の値を捨てる → 7
>>> run([["PUSH",7],["PUSH",99],["POP",0]])
7
# テスト: 単一 PUSH
>>> run([["PUSH",42]])
42
```

```yui
標準ライブラリを使う
# 点数から成績 (A/B/C/D/F) を決める多段分岐
# yui には elif が無いので、そうでなければ の中に もし をネストして表現する。
# A: 90 以上, B: 80 以上, C: 70 以上, D: 60 以上, F: それ未満

grade=入力scoreに対し{
   result="F"
   もしscoreが90以上ならば{
      result="A"
   }
   そうでなければ{
      もしscoreが80以上ならば{
         result="B"
      }
      そうでなければ{
         もしscoreが70以上ならば{
            result="C"
         }
         そうでなければ{
            もしscoreが60以上ならば{
               result="D"
            }
         }
      }
   }
   resultが答え
}

>>> grade(100)
"A"
>>> grade(95)
"A"
>>> grade(89)
"B"
>>> grade(70)
"C"
>>> grade(65)
"D"
>>> grade(59)
"F"
>>> grade(0)
"F"

# 合否判定 (以外 の例: F 以外なら合格)
passed=入力scoreに対し{
   g=grade(score)
   もしgが"F"以外ならば{
      真が答え
   }
   そうでなければ{
      偽が答え
   }
}

>>> passed(75)
真
>>> passed(40)
偽
```

```yui
標準ライブラリを使う
# yui の関数は値を 1 つしか返せないが、複数の結果は次の 3 通りで表現できる
#
#   1. **配列で返す**  [a, b]    取り出すときは r[0], r[1]
#   2. **オブジェクトで返す**  {"q": ..., "r": ...}   取り出すときは r["q"]
#   3. **暗黙の戻り値**  関数本体の最後に が答え を書かないと、
#                        ローカル変数をまとめたオブジェクトが返る
#                        (コンストラクタ風の使い方に向く)

# === 1. 配列で返す: 商と余りを同時に返す ===
divmod_array=入力a,bに対し{
   q=商(a,b)
   r=剰余(a,b)
   [q,r]が答え
}

# === 2. オブジェクトで返す: 配列の統計値をまとめて返す ===
stats=入力arrに対し{
   lo=arr[0]
   hi=arr[0]
   total=0
   n=arrの大きさ
   i=0
   n回くり返す{
      x=arr[i]
      もしxがloより小さいならば{ lo=x }
      もしxがhiより大きいならば{ hi=x }
      total=和(total,x)
      iを増やす
   }
   {"min":lo,"max":hi,"sum":total,"count":n}が答え
}

# === 3. 暗黙の戻り値: コンストラクタ風 ===
# が答え を書かずに関数を終えると、ローカル変数 (パラメータ含む) を
# まとめたオブジェクトが返る。小さな構造体を作るときに便利。
point=入力x,yに対し{
   # 何も書かない → {x, y} が返る
}

# === テスト 1: 配列で返す ===
>>> divmod_array(17,5)
[3,2]
>>> divmod_array(20,4)
[5,0]
# 取り出して使う
qr=divmod_array(17,5)
>>> qr[0]
3
>>> qr[1]
2

# === テスト 2: オブジェクトで返す ===
s=stats([3,1,4,1,5,9,2,6])
>>> s["min"]
1
>>> s["max"]
9
>>> s["sum"]
31
>>> s["count"]
8

# === テスト 3: 暗黙の戻り値 ===
P=point(3,5)
>>> P["x"]
3
>>> P["y"]
5
```

```yui
標準ライブラリを使う
# 複利計算
#
# 元本 P を年利 r で n 年運用したときの最終金額は、いわゆる複利の公式で
#   A は P と (1+r) の n 乗の積
# になる。yui には べき乗 が無いので、毎年 balance を (1+r) 倍するループで実装する。
#
# 利率 r は小数で渡す (例: 5% なら 0.05)。
# 結果は小数で返る。

# === 1. n 年後の元利合計 ===
compound=入力P,r,nに対し{
   balance=小数化(P)
   factor=和(小数化(1),r)
   n回くり返す{
      balance=積(balance,factor)
   }
   balanceが答え
}

# === 2. 元本が 2 倍になるのに何年かかるか (年単位の切り上げ) ===
# 利率 r で運用したとき、balance が初めて元本の 2 倍以上になる年を返す
years_to_double=入力rに対し{
   balance=小数化(1)
   factor=和(小数化(1),r)
   n=0
   # 念のため上限 1000 年でガード
   1000回くり返す{
      もしbalanceが2以上ならば{
         くり返しを抜ける
      }
      balance=積(balance,factor)
      nを増やす
   }
   nが答え
}

# === テスト: 1 年目 ===
>>> compound(1000,0.05,1)
1050.000000
# === テスト: 2 年目 (1000 × 1.05 × 1.05) ===
>>> compound(1000,0.05,2)
1102.500000
# === テスト: 3 年目 (1.10 を 3 回かける) ===
>>> compound(1000,0.10,3)
1331.000000
# === テスト: 利率 0 なら元本のまま ===
>>> compound(100,0.0,5)
100.000000
# === テスト: 0 年なら元本のまま ===
>>> compound(1000,0.05,0)
1000.000000

# === テスト: 年利 10 パーセントなら、8 年目に元本が 2 倍を超える ===
>>> years_to_double(0.10)
8
# === テスト: 年利 7 パーセントなら 11 年 ===
>>> years_to_double(0.07)
11
# === テスト: 年利 5 パーセントなら 15 年 (いわゆる 72 の法則) ===
>>> years_to_double(0.05)
15
```

```yui
標準ライブラリを使う
```
```yui
# 「のいずれか」「のいずれでもない」による集合検索

menu=["カレー","ラーメン","寿司","そば"]

# のいずれか: 要素が含まれるかを調べる
check=入力itemに対し{
   もしitemがmenuのいずれかならば{
      "あります"が答え
   }
   そうでなければ{
      "ありません"が答え
   }
}

>>> check("寿司")
"あります"
>>> check("ピザ")
```

```yui
# サイコロを N 回振って、1〜6 の各目が出た回数を数える
# 乱数() は 0 以上 1 未満の小数を返すので、6 倍して 整数化 すれば 0..5 の整数
```

```yui
標準ライブラリを使う
# ROT13: アルファベットを 13 文字ぶん回転させる古典暗号
# 文字列は内部的に文字コード配列なので 1 文字ずつコード値を取り出して変換する
# 文字コード: 'A'=65, 'Z'=90, 'a'=97, 'z'=122
# ROT13 は自己逆 (2 回適用すると元に戻る) なので暗号化と復号化は同じ関数で済む

rot13=入力sに対し{
   result=""
   n=sの大きさ
   i=0
   n回くり返す{
      c=s[i]
      nc=c
      # 大文字 A..Z
      もしcが65以上ならば{
         もしcが90以下ならば{
            offset=差(c,65)
            shifted=剰余(和(offset,13),26)
            nc=和(shifted,65)
         }
      }
      # 小文字 a..z
      もしcが97以上ならば{
         もしcが122以下ならば{
            offset=差(c,97)
            shifted=剰余(和(offset,13),26)
            nc=和(shifted,97)
         }
      }
      resultにncを追加する
      iを増やす
   }
   resultが答え
}

# テスト: 大文字の基本ケース  ABC → NOP
>>> rot13("ABC")
"NOP"
# テスト: 小文字の基本ケース  xyz → klm (折り返し)
>>> rot13("xyz")
"klm"
# テスト: 混在 + 記号 + 数字 (アルファベット以外はそのまま)
>>> rot13("Hello, World!")
"Uryyb, Jbeyq!"
# テスト: 自己逆性  rot13(rot13(x)) == x
>>> rot13("Uryyb, Jbeyq!")
"Hello, World!"
# テスト: 数字・記号は不変
>>> rot13("123 !?")
"123 !?"
# テスト: 空文字列
>>> rot13("")
""
```
