# Yui ハマりどころ集

## 1. 『もし』の条件に真偽値を直接置けない

### 症状

`もし` の条件部に、真偽値を返す式をそのまま置くとパース失敗する。

```
もし整数判定(x)ならば{ ... }       ← FAIL
もしokならば{ ... }                ← FAIL  (ok は bool 変数)
```

### 回避策

`もし A が B <接尾辞> ならば` の中置比較で、真偽値なら `真` と比較する。

```
もし整数判定(x)が真ならば{ ... }
もしokが真ならば{ ... }
```

### ルール

yui の `もし` は **中置比較必須**。python の `if x:` の感覚で真偽値を
直接書かず、必ず `が 真 ならば` / `が 偽 ならば` を明示する。

## 2. 比較の接尾辞は 5 種類だけ

### 症状

日本語として自然な比較語を書いてもパース失敗する。

```
もしnが2未満ならば{ ... }             ← FAIL  (「未満」は未対応)
もしaがbでないならば{ ... }            ← FAIL  (「でない」は未対応)
もしsが"x"を含むならば{ ... }          ← FAIL  (該当する接尾辞なし)
もしaがbと等しくないならば{ ... }       ← FAIL
```

### 回避策

使える接尾辞は以下の 5 種類のみ。

| 比較 | 接尾辞 |
|------|--------|
| `==` | (なし) |
| `!=` | `以外` |
| `<`  | `より小さい` |
| `>`  | `より大きい` |
| `<=` | `以下` |
| `>=` | `以上` |

```
もしnが2より小さいならば{ ... }
もしaがb以外ならば{ ... }
```

### ルール

「未満」「でない」「異なる」「を含む」はすべて自然な日本語に見えるが
yui では使えない。必ず上の表の接尾辞に置き換える。
「を含む」は組み込みがないので、自前で走査する必要がある。

## 3. 無限ループ構文はない

### 症状

回数を省略した `くり返す { ... }` はパース失敗する。

```
くり返す{
   もし条件ならば{ くり返しを抜ける }
   ...
}                                       ← FAIL
```

`i が x 以下の間くり返す` のような while 相当もない。

### 回避策

十分大きな上限を置き、条件を満たしたら `くり返しを抜ける` で脱出する。

```
100000回くり返す{
   もし条件ならば{ くり返しを抜ける }
   ...
}
```

### ルール

反復構文は `N回くり返す` のみ。`N` は必ず整数リテラルまたは変数で書き、
終了条件が事前に分からないときは **上限 + 早期脱出** で書く。

## 4. 非存在組み込み関数を呼ばない

### 症状

python や javascript 標準の関数名をそのまま呼ぶとパース後に未定義エラー。

```
arrをソートする                          ← FAIL
並べ替え(arr)                           ← FAIL
分割(s, ",")                            ← FAIL
"a,b"を","で分割                        ← FAIL
辞書のキー列                            ← FAIL
md5(s)                                  ← FAIL
切り上げ(x)                             ← FAIL
天井(x)                                 ← FAIL
べき乗(a,b)                             ← FAIL
除算(a,b)                               ← FAIL
approx_zero(...)                        ← FAIL
rshift(x, n)                            ← FAIL
文字列補間(...)                          ← FAIL
```

### 回避策

yui の組み込みは以下の 20 種のみ。それ以外は自前で実装する。

| 種別 | 関数 |
|------|------|
| 算術 | `和`, `差`, `積`, `商`, `剰余`, `絶対値`, `平方根`, `乱数` |
| 集約 | `最大値`, `最小値` |
| 変換 | `整数化`, `小数化`, `文字列化`, `配列化` |
| 型判定 | `整数判定`, `小数判定`, `文字列判定`, `配列判定`, `オブジェクト判定`, `ブール判定` |

ソート・反転・部分文字列検索・分割・切り上げ・べき乗・md5・ハッシュ関数
などは組み込みにない。必要なら関数の中で自前ループで書く。

```
# 並べ替えは二重ループの隣接交換で書く
# 分割は文字コード配列の走査で書く
```

### ルール

「python にあるから yui にもあるはず」と考えない。組み込みにない処理は
**関数内にインライン展開** して書く。

## 5. 配列追加は `配列 に 値 を追加する` の語順

### 症状

助詞が逆の語順で書いてしまう。

```
stackをargに追加する                    ← FAIL  (語順が逆)
```

### 回避策

```
stackにargを追加する
```

### ルール

`配列 に 値 を追加する` で固定。日本語としてはどちらの語順も自然に読めるが、
yui では語順が固定されている。

## 6. 判定関数の戻り値は `1` / `0` (整数)

### 症状

判定関数が `真が答え` / `偽が答え` を返すと、期待値 `1`/`0` と不一致になる。

```
is_prime=入力nに対し{
   ...
   真が答え                              ← 期待値は 1
}
```

### 回避策

判定結果は整数 `1` / `0` で返す。

```
is_prime=入力nに対し{
   result=0
   もし ... ならば{ result=1 }
   resultが答え
}
```

### ルール

「〜かどうか」「含まれているか」「等しいか」など **yes/no を返す関数は
必ず 1/0 の整数で返す**。`真` `偽` (真偽値) を返すと数値比較と不一致になる。

複数の判定を組み合わせるときは `積(a,b)` (AND)、`和(a,b)` が 0 より大きいか
で判定 (OR)。

## 7. プロンプト内の例示関数を直接呼び出さない

### 症状

プロンプトに `bubble_sort` や `reverse` や `find` や `unique_chars` のような
サンプル関数があっても、生成する関数からそれらを呼び出すと未定義エラーになる。

```
median=入力lに対し{
   sorted=bubble_sort(l)                ← FAIL  (定義を含めていないと未定義)
   ...
}
```

### 回避策

再利用したい処理は関数の中にインライン展開する。

```
median=入力lに対し{
   # ここに bubble_sort の中身を直接書く
   n=lの大きさ
   sorted=[]
   i=0
   n回くり返す{ sortedにl[i]を追加する; iを増やす }
   i=0
   n回くり返す{
      j=0
      差(差(n,i),1)回くり返す{
         a=sorted[j]; k=和(j,1); b=sorted[k]
         もしaがbより大きいならば{ sorted[j]=b; sorted[k]=a }
         jを増やす
      }
      iを増やす
   }
   sorted[商(n,2)]が答え
}
```

### ルール

プロンプトの例示は「参考」であって「共有ライブラリ」ではない。
生成する関数の中で何らかの処理を使いたい場合は **その関数内で再実装** する。
特にソート・反転・部分文字列検索はこのミスが起きやすい。

## 8. 算術記号 `+ - * / %` は使えない

### 症状

```
和=a+b                                   ← FAIL
```

### 回避策

標準ライブラリ関数を使う。

| やりたいこと | 書き方 |
|--------------|--------|
| `a + b`      | `和(a,b)` |
| `a - b`      | `差(a,b)` |
| `a * b`      | `積(a,b)` |
| `a / b` (整数商) | `商(a,b)` |
| `a % b`      | `剰余(a,b)` |

ループのインデックス計算 (`i+1`, `n-1`) も同様で、`和(i,1)` / `差(n,1)` と書く。

### ルール

算術演算子は一切使えない。必ず関数呼び出し形式で書く。

## 9. 複数値を返すときは配列 `[a, b]`

### 症状

タプル相当の戻り値をオブジェクトで返すと期待値と不一致になる。

```
sum_product=入力arrに対し{
   ...
   {"sum":s, "product":p}が答え          ← 期待値は [s, p]
}
```

### 回避策

配列で返す。受け取り側は `r[0]`, `r[1]` で取り出す。

```
sum_product=入力arrに対し{
   ...
   [s, p]が答え
}
```

### ルール

複数の値を返したいときは **常に配列 `[a, b, ...]` で返す**。
オブジェクト (`{"key": v}`) で返すと評価のタプル期待と一致しない。
プロンプト中のサンプルには配列/オブジェクト両方の返し方が出てくるが、
本ベンチマークでは配列を選ぶ。

## 10. 文字列 `"0"` と整数 `0` は別物

### 症状

数値を期待する関数に文字列を渡すと型エラー。

```
change_base(x, "2")                     ← FAIL  (基数に文字列)
decimal_to_binary("0")                  ← FAIL
```

### 回避策

- `"42"` → 整数: `整数化("42")`
- 整数 → `"42"`: `文字列化(42)`
- 数字1文字コード (`'0'`=48, `'9'`=57) → 整数: `差(c, 48)`
- 整数 (0..9) → 数字1文字コード: `和(n, 48)`

### ルール

yui の文字列は内部的に文字コードの配列。リテラル `"0"` と整数 `0` は別の型
なので、どちらを扱っているか常に意識する。

```yui
標準ライブラリを使う
# 簡易スタックマシン
# 命令は [opcode, operand] の形の配列で表す。operand が不要な命令は 0 を入れる。
#
# 対応命令:
#   ["PUSH", n] -- スタックに n を積む
#   ["POP",  0] -- スタック頂上を捨てる
#   ["DUP",  0] -- スタック頂上を複製する
#   ["ADD",  0] -- 上 2 つを取り出して a+b を積む
#   ["SUB",  0] -- 上 2 つを取り出して a-b を積む
#   ["MUL",  0] -- 上 2 つを取り出して a*b を積む
#
# プログラム実行後、スタックの頂上を答えとして返す。
# (Yui の配列は要素削除ができないので、論理的なスタックの大きさを top で管理し、
#  pop は top を減らすだけにする)

run=入力programに対し{
   stack=[]
   top=0
   n=programの大きさ
   i=0
   n回くり返す{
      instr=program[i]
      op=instr[0]
      arg=instr[1]

      もしopが"PUSH"ならば{
         # 論理 top が配列実長より小さければ上書き、そうでなければ末尾に追加
         sz=stackの大きさ
         もしtopがszより小さいならば{
            stack[top]=arg
         }
         そうでなければ{
            stackにargを追加する
         }
         topを増やす
      }
      もしopが"POP"ならば{
         topを減らす
      }
      もしopが"DUP"ならば{
         t=差(top,1)
         x=stack[t]
         sz=stackの大きさ
         もしtopがszより小さいならば{
            stack[top]=x
         }
         そうでなければ{
            stackにxを追加する
         }
         topを増やす
      }
      もしopが"ADD"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=和(a,b)
         topを増やす
      }
      もしopが"SUB"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=差(a,b)
         topを増やす
      }
      もしopが"MUL"ならば{
         topを減らす
         b=stack[top]
         topを減らす
         a=stack[top]
         stack[top]=積(a,b)
         topを増やす
      }
      iを増やす
   }
   # スタックの頂上を結果として返す
   topを減らす
   stack[top]が答え
}

# テスト: 3 + 4 = 7
>>> run([["PUSH",3],["PUSH",4],["ADD",0]])
7
# テスト: (3 + 4) * 2 = 14
>>> run([["PUSH",3],["PUSH",4],["ADD",0],["PUSH",2],["MUL",0]])
14
# テスト: 10 - (2 + 3) = 5
>>> run([["PUSH",10],["PUSH",2],["PUSH",3],["ADD",0],["SUB",0]])
5
# テスト: DUP で 5 を複製して掛ける → 25
>>> run([["PUSH",5],["DUP",0],["MUL",0]])
25
# テスト: POP で途中の値を捨てる → 7
>>> run([["PUSH",7],["PUSH",99],["POP",0]])
7
# テスト: 単一 PUSH
>>> run([["PUSH",42]])
42
```

```yui
標準ライブラリを使う
```
```yui
# yui の関数は値を 1 つしか返せないが、複数の結果は次の 3 通りで表現できる
#
#   1. **配列で返す**  [a, b]    取り出すときは r[0], r[1]
#   2. **オブジェクトで返す**  {"q": ..., "r": ...}   取り出すときは r["q"]

   r=剰余(a,b)

}

}

# まとめたオブジェクトが返る。小さな構造体を作るときに便利。
point=入力x,yに対し{
   # 何も書かない → {x, y} が返る
}

# === テスト 1: 配列で返す ===

>>> divmod_array(20,4)
```

```yui
標準ライブラリを使う
# 複利計算
#
# 元本 P を年利 r で n 年運用したときの最終金額は、いわゆる複利の公式で
#   A は P と (1+r) の n 乗の積
# になる。yui には べき乗 が無いので、毎年 balance を (1+r) 倍するループで実装する。
#
# 利率 r は小数で渡す (例: 5% なら 0.05)。
# 結果は小数で返る。

# === 1. n 年後の元利合計 ===
compound=入力P,r,nに対し{
   balance=小数化(P)
   factor=和(小数化(1),r)
   n回くり返す{
      balance=積(balance,factor)
   }
   balanceが答え
}

# === 2. 元本が 2 倍になるのに何年かかるか (年単位の切り上げ) ===
# 利率 r で運用したとき、balance が初めて元本の 2 倍以上になる年を返す
years_to_double=入力rに対し{
   balance=小数化(1)
   factor=和(小数化(1),r)
   n=0
   # 念のため上限 1000 年でガード
   1000回くり返す{
      もしbalanceが2以上ならば{
         くり返しを抜ける
      }
      balance=積(balance,factor)
      nを増やす
   }
   nが答え
}

# === テスト: 1 年目 ===
>>> compound(1000,0.05,1)
1050.000000
# === テスト: 2 年目 (1000 × 1.05 × 1.05) ===
>>> compound(1000,0.05,2)
1102.500000
# === テスト: 3 年目 (1.10 を 3 回かける) ===
>>> compound(1000,0.10,3)
1331.000000
# === テスト: 利率 0 なら元本のまま ===
>>> compound(100,0.0,5)
100.000000
# === テスト: 0 年なら元本のまま ===
>>> compound(1000,0.05,0)
1000.000000

# === テスト: 年利 10 パーセントなら、8 年目に元本が 2 倍を超える ===
>>> years_to_double(0.10)
8
# === テスト: 年利 7 パーセントなら 11 年 ===
>>> years_to_double(0.07)
11
# === テスト: 年利 5 パーセントなら 15 年 (いわゆる 72 の法則) ===
>>> years_to_double(0.05)
15
```

```yui

# 何もしない のデモ
# 条件分岐で「このケースでは敢えて何もしない」ことを明示したいときに使う。
# 空の { } だと構文的に書けない場面のプレースホルダにも便利。

sign=入力nに対し{
   result=""
   もしnが0ならば{
      何もしない
   }
   そうでなければ{
      もしnが0より小さいならば{
         result="負"
      }
      そうでなければ{
         result="正"
      }
   }
   resultが答え
}

>>> sign(0)
```

```yui
標準ライブラリを使う
# ROT13: アルファベットを 13 文字ぶん回転させる古典暗号
# 文字列は内部的に文字コード配列なので 1 文字ずつコード値を取り出して変換する
# 文字コード: 'A'=65, 'Z'=90, 'a'=97, 'z'=122
# ROT13 は自己逆 (2 回適用すると元に戻る) なので暗号化と復号化は同じ関数で済む

rot13=入力sに対し{
   result=""
   n=sの大きさ
   i=0
   n回くり返す{
      c=s[i]
      nc=c
      # 大文字 A..Z
      もしcが65以上ならば{
         もしcが90以下ならば{
            offset=差(c,65)
            shifted=剰余(和(offset,13),26)
            nc=和(shifted,65)
         }
      }
      # 小文字 a..z
      もしcが97以上ならば{
         もしcが122以下ならば{
            offset=差(c,97)
            shifted=剰余(和(offset,13),26)
            nc=和(shifted,97)
         }
      }
      resultにncを追加する
      iを増やす
   }
   resultが答え
}

# テスト: 大文字の基本ケース  ABC → NOP
>>> rot13("ABC")
"NOP"
# テスト: 小文字の基本ケース  xyz → klm (折り返し)
>>> rot13("xyz")
"klm"
# テスト: 混在 + 記号 + 数字 (アルファベット以外はそのまま)
>>> rot13("Hello, World!")
"Uryyb, Jbeyq!"
# テスト: 自己逆性  rot13(rot13(x)) == x
>>> rot13("Uryyb, Jbeyq!")
"Hello, World!"
# テスト: 数字・記号は不変
>>> rot13("123 !?")
"123 !?"
# テスト: 空文字列
>>> rot13("")
""
```
